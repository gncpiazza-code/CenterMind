§5   Leyenda y conceptos

⚡
TanStack Query

Librería de server-state para React. Maneja fetch, cache, deduplicación, stale-while-revalidate y mutaciones optimistas. Los datos del servidor no viven en estado React local — viven en el QueryClient y se comparten entre componentes automáticamente.

🐻
Zustand

Librería de client-state minimalista. Solo para estado de UI puro: qué tab está activo, qué vendedores son visibles en el mapa, índice de foto actual. No almacena datos del servidor.

✅
Módulo completo

100% del fetching usa TanStack Query. Cero duplicación de requests. Stale-while-revalidate activo: datos instantáneos del cache mientras se revalida en background.

⚠️
Migración parcial

Parte del módulo usa TanStack Query (ej. mapa de rutas) y otra parte sigue con useState + fetch manual. La deduplicación no cubre todos los sub-componentes.

❌
Legacy / Pendiente

Fetching manual con useState + useEffect. Sin cache entre navegaciones, sin deduplicación. Cada render nuevo del componente dispara requests al servidor.

💾
Persist (localStorage)

useSupervisionStore persiste en localStorage con serialización custom de Sets (no serializable en JSON nativo). Al recargar la página, el supervisor recupera la sucursal seleccionada y la visibilidad de capas del mapa.
