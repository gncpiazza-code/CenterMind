🔀 Interceptor de Franquiciados (Phase 1 — NUEVO)

Solución estructural para vendedores que operan bajo cuentas delegadas (ej. Ivan Soto). Las exhibiciones se reatribuyen al vendedor real en el momento de registro.

📱 Soto sube foto
uploader_id = soto_tuid
→
🔍 lookup_soto_intercept()
DB: matcheo_rutas_excepciones
→
✅ Match encontrado
real_tuid = monchi / jorge
→
📝 registrar_exhibicion()
vendedor_id = real_tuid
▶ Ver SQL — Crear tabla matcheo_rutas_excepciones 
CREATE TABLE IF NOT EXISTS matcheo_rutas_excepciones (
    id                            SERIAL PRIMARY KEY,
    id_distribuidor               INTEGER NOT NULL,
    telegram_user_id_franquiciado BIGINT  NOT NULL,
    id_cliente_erp                TEXT    NOT NULL,
    telegram_user_id_real         BIGINT  NOT NULL,
    nombre_vendedor_real          TEXT,
    ruta_inferida                 TEXT,
    confianza                     TEXT DEFAULT 'alta',
    created_at                    TIMESTAMPTZ DEFAULT now(),
    UNIQUE (id_distribuidor, telegram_user_id_franquiciado, id_cliente_erp)
);

🔒 Auditoría de Seguridad — Hallazgos Phase 2

✅
Credenciales en código fuente
Sin credenciales hardcodeadas encontradas en api.py, bot_worker.py ni services/. Todas las claves se leen de variables de entorno.
⚠️
Paths Windows hardcodeados en scripts de audit
~30 scripts de diagnóstico usan load_dotenv("c:\Users\cigar\..."). Estos fallan silenciosamente en Mac/Linux. Scripts afectados son de diagnóstico/auditoría (no producción). Corrección: reemplazar con load_dotenv(Path(__file__).parent / ".env").
✅
SQL Injection (api.py)
Todos los queries usan el client Supabase Python con parámetros tipados. No se encontró concatenación dinámica de SQL. Las RPCs usan parámetros nombrados (p_dist_id, etc.).
✅
Supabase Storage — Content-Type
SupabaseUploader.upload() fuerza content-type: image/jpeg en todas las subidas. No se aceptan uploads arbitrarios desde el bot.
✅
Autenticación — JWT + API Key
verify_auth() valida ambos métodos. check_dist_permission() previene acceso cruzado entre tenants. Algoritmo HS256 con expiración 8h.
⚠️
Path Traversal en nro_cliente (Telegram)
El nro_cliente ingresado por el vendedor se usa en el filename de Storage ({nro_cliente}_{clean_code}_{ts}.jpg). El storage path usa safe_dist filtrado, pero nro_cliente no se sanitiza antes de entrar al filename. Recomendación: aplicar re.sub(r'[^a-zA-Z0-9_-]', '_', nro_cliente) en el filename.
✅
Tokens JWT en JWT — sin exposición
No se encontraron JWT hardcodeados (patrón eyJ...) en ningún archivo .py del repositorio.
